## Repository Overview

```
Novelty/
├── android/              # Android platform code
├── ios/                  # iOS platform code
├── linux/                # Linux platform code
├── macos/                # macOS platform code
├── windows/              # Windows platform code
├── web/                  # Web platform code (experimental)
├── lib/                  # Main application code (Dart/Flutter)
├── test/                 # Unit and widget tests
├── packages/             # Local packages (workspace)
├── assets/               # Static assets (fonts, icons)
├── docs/                 # This documentation
├── .github/              # GitHub Actions workflows
├── pubspec.yaml          # Dependencies and metadata
├── mise.toml             # Development tasks and tools
├── lefthook.yml          # Git hooks configuration
└── analysis_options.yaml # Lint rules and analyzer config
```

## Main Application Code (`lib/`)

The `lib/` directory contains all the Dart application code:

```
lib/
├── main.dart             # Application entry point
├── database/             # Database layer (Drift)
├── domain/               # Domain models and state
├── models/               # Data models
├── providers/            # Riverpod providers
├── repositories/         # Repository pattern
├── router/               # Routing configuration
├── screens/              # UI screens/pages
├── services/             # External services
├── utils/                # Utility functions
└── widgets/              # Reusable UI components
```

### Entry Point

<Accordion title="main.dart">
  **Location**: `lib/main.dart`
  
  **Purpose**: Application bootstrap and initialization
  
  **Contents**:
  - `main()` function - Entry point
  - `MyApp` widget - Root widget with:
    - `ProviderScope` for Riverpod
    - `DynamicColorBuilder` for Material You theming
    - `MaterialApp.router` with GoRouter configuration
  
  **Key Code**:
  ```dart
  Future<void> main() async {
    WidgetsFlutterBinding.ensureInitialized();
    runApp(const ProviderScope(child: MyApp()));
  }
  ```
</Accordion>

## Database Layer

<Tabs>
  <Tab title="Structure">
    ```
    database/
    ├── database.dart            # Main database class
    ├── database.g.dart          # Generated code (Drift)
    ├── database_providers.dart  # Riverpod providers for DB
    └── database_providers.g.dart
    ```
  </Tab>
  
  <Tab title="Tables">
    **Defined in** `database.dart`:
    
    | Table Class | Table Name | Purpose |
    |-------------|------------|----------|
    | `Novels` | `novels` | Master novel information |
    | `LibraryEntries` | `library_entries` | User's library |
    | `ReadingHistory` | `reading_history` | Reading progress |
    | `EpisodeEntities` | `episodes` | Episode content cache |
  </Tab>
  
  <Tab title="Key Classes">
    - `AppDatabase` - Main database class
    - `ContentConverter` - JSON converter for episode content
    - `HistoryData` - DTO for history with joined data
    - `EpisodeSearchResult` - DTO for episode search results
  </Tab>
  
  <Tab title="Providers">
    **Defined in** `database_providers.dart`:
    
    ```dart
    final databaseProvider = Provider((ref) => AppDatabase());
    
    final libraryNovelsProvider = StreamProvider((ref) {
      final db = ref.watch(databaseProvider);
      return db.watchLibraryNovels();
    });
    
    final historyProvider = StreamProvider((ref) {
      final db = ref.watch(databaseProvider);
      return db.watchHistory();
    });
    ```
  </Tab>
</Tabs>

## Services

```
services/
├── api_service.dart       # なろう API client
├── api_service.g.dart
└── backup_service.dart    # Data export/import
```

<AccordionGroup>
  <Accordion title="ApiService">
    **Base URL**: `https://api.syosetu.com`
    
    **Endpoints**:
    - `/novelapi/api/` - Novel information and search
    - HTML scraping for episode content
    
    **Methods**:
    ```dart
    Future<List<NovelInfo>> fetchNovelInfo(String ncode);
    Future<List<NovelInfo>> fetchRanking({String type, String? genre});
    Future<List<NovelInfo>> searchNovels(NovelSearchQuery query);
    Future<List<NovelContentElement>> fetchEpisodeContent(String url);
    ```
  </Accordion>
  
  <Accordion title="BackupService">
    **Purpose**: Export and import library data
    
    **Format**: ZIP archive containing:
    - `novels.json` - Novel metadata
    - `library.json` - Library entries
    - `history.json` - Reading history
    - `episodes.json` - Downloaded episodes
    
    **Methods**:
    ```dart
    Future<File> exportBackup();
    Future<void> importBackup(File backupFile);
    ```
  </Accordion>
</AccordionGroup>

## Generated Files

<Warning>
  **Never edit generated files manually!**
  
  These files are regenerated by `build_runner`:
  - `*.g.dart` - Riverpod, Drift, JSON serialization
  - `*.freezed.dart` - Freezed data classes
  - `*.mocks.dart` - Mockito mocks (tests)
</Warning>

**Regenerate**:

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

## Finding What You Need

<AccordionGroup>
  <Accordion title="Adding a new feature">
    1. **Model**: Create data class in `lib/models/`
    2. **Service**: Add API call to `lib/services/api_service.dart`
    3. **Repository**: Add business logic to `lib/repositories/`
    4. **Provider**: Create state provider in `lib/providers/` or `lib/domain/`
    5. **Screen**: Build UI in `lib/screens/`
    6. **Route**: Register in `lib/router/router.dart`
    7. **Tests**: Add tests in `test/`
  </Accordion>
  
  <Accordion title="Fixing a bug">
    - **UI bug**: Check `lib/screens/` and `lib/widgets/`
    - **Data bug**: Check `lib/database/` and `lib/repositories/`
    - **API bug**: Check `lib/services/api_service.dart`
    - **State bug**: Check `lib/providers/` and `lib/domain/`
  </Accordion>
  
  <Accordion title="Improving performance">
    - **Database queries**: `lib/database/database.dart`
    - **Widget rebuilds**: `lib/screens/` and providers
    - **API calls**: `lib/services/api_service.dart` caching
    - **Parsing**: `packages/narou_parser/`
  </Accordion>
</AccordionGroup>
