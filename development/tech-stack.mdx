## State Management

### Riverpod Ecosystem

Novelty uses the Riverpod family of packages for comprehensive state management:

| Package | Version | Purpose |
|---------|---------|--------|
| `riverpod` | ^3.0.3 | Core state management primitives |
| `flutter_riverpod` | ^3.0.3 | Flutter integration for Riverpod |
| `hooks_riverpod` | ^3.0.3 | Riverpod + Flutter Hooks integration |
| `riverpod_annotation` | ^3.0.3 | Annotations for code generation |
| `riverpod_generator` | ^3.0.0 | Code generator for providers |
| `riverpod_lint` | ^3.0.0 | Lint rules for best practices |

**Why Riverpod?**

<AccordionGroup>
  <Accordion title="Type Safety">
    Compile-time safety with code generation. No runtime errors from mistyped provider names.
  </Accordion>
  
  <Accordion title="Testability">
    Easy to override providers in tests. No need for complex mocking setups.
  </Accordion>
  
  <Accordion title="Performance">
    Automatic dependency tracking and caching. Only rebuilds what's necessary.
  </Accordion>
  
  <Accordion title="Developer Experience">
    Full hot reload support, clear error messages, and excellent debugging tools.
  </Accordion>
</AccordionGroup>

## Routing & Navigation

### GoRouter

- **Package**: `go_router` ^17.0.0
- **Features**:
  - Declarative routing
  - Deep linking support
  - Type-safe route parameters
  - Nested navigation with `StatefulShellRoute`

**Route Configuration**:

```dart
// Main navigation shell with 4 tabs
StatefulShellRoute.indexedStack(
  branches: [
    StatefulShellBranch(routes: [LibraryPage]),  // /
    StatefulShellBranch(routes: [ExplorePage]),  // /explore
    StatefulShellBranch(routes: [HistoryPage]),  // /history
    StatefulShellBranch(routes: [MorePage]),     // /more
  ],
)

// Full-screen routes
GoRoute(path: '/novel/:ncode')  // Novel detail
GoRoute(path: '/novel/:ncode/:episode')  // Reader
```

## Data Serialization

### Freezed

- **Package**: `freezed` ^3.1.0
- **Annotation**: `freezed_annotation` ^3.1.0

**Features**:
- Immutable data classes
- Union types
- Copy-with methods
- Pattern matching

```dart
@freezed
class NovelInfo with _$NovelInfo {
  const factory NovelInfo({
    required String ncode,
    String? title,
    String? writer,
  }) = _NovelInfo;
  
  factory NovelInfo.fromJson(Map<String, dynamic> json) 
    => _$NovelInfoFromJson(json);
}
```

## HTML Parsing

### HTML Package

- **Package**: `html` ^0.15.6
- **Purpose**: Parse HTML responses from "なろう小説" website

### Narou Parser

- **Package**: Custom local package (`packages/narou_parser`)
- **Purpose**: Convert "なろう" HTML to structured content

**Parsers Available**:

| Parser | Speed | Method |
|--------|-------|--------|
| Stable | Baseline | DOM-based (package:html) |
| Fast | ~10x faster | Regex-based |
| Ultra | ~100x faster | String scanning |

## Custom Packages

## Build & CI/CD

### Task Runner

- **mise**: ^2024 - Development environment manager
- **Configured in**: `mise.toml`

**Available Tasks**:

```bash
mise run setup      # Setup Android/Flutter environment
mise run get        # Install dependencies
mise run codegen    # Run code generation
mise run run        # Run the app
mise run test       # Run tests
mise run build      # Build release APK/AAB
mise run format     # Format code
mise run analyze    # Static analysis
```
